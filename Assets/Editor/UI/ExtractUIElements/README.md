# ExtractUIComponent

> Unity Editor 插件 — 通过节点标记符自动提取 UIForm 预制件中的组件引用，生成 `partial` 字段声明，并在脚本编译后自动完成组件绑定。

---

## 功能概览

- **节点标记驱动**：在 Hierarchy 节点名前加一个 ASCII 字符（默认 `@` `#` `=` `-`），精确控制哪些节点的组件参与导出
- **可视化预览**：导出前在树形界面预览每个节点的组件列表，支持逐项勾选调整
- **自动代码生成**：生成与主脚本同目录的 `{ClassName}.Components.cs`，保持相同命名空间、自动补全 `using` 和 `partial` 关键字
- **编译后自动绑定**：脚本编译完成后由 `[InitializeOnLoad]` 监听器自动完成字段赋值并保存预制件
- **互斥规则**：通过矩阵面板配置同节点上多个组件之间的互斥关系，避免冗余字段
- **优先级排序**：检索规则支持拖拽排序，顺序即优先级，高优先级规则在互斥判定中胜出
- **标记符可自定义**：4 个标记均为单字符 ASCII，可在导出设置面板中修改并一键重置

---

## 安装

将 `Assets/Editor/UI/ExtractUIElements/` 目录整体复制到项目的 `Editor` 文件夹下即可。插件依赖：

- Unity 2021.3+（使用 `GlobalObjectId` API）
- `UnityEditorInternal`（`ReorderableList`，已内置）

## 快速上手

### 第一步：初始化配置

菜单 **Window › 界面工具 › 初始化**，填写：

| 字段 | 说明 | 示例 |
|---|---|---|
| UIForm Prefab 根路径 | 预制件所在目录（可多条） | `Assets/GameAssets/UIForms/` |
| UIForm 基类全称 | 含命名空间的完整类名 | `UnityGameFramework.Runtime.UIFormLogic` |

> 已有配置时可直接点击 **更新** 修改路径，不会影响检索规则和互斥规则。

### 第二步：配置检索规则

菜单 **Window › 界面工具 › 检索规则**

配置哪些组件类型参与导出及对应的字段前缀。列表顺序即优先级（上低下高），可拖拽调整。

| 是否检索 | 组件类（完整类名） | 字段开头 |
|:---:|---|---|
| ✓ | `UnityEngine.UI.Button` | `Btn` |
| ✓ | `TMPro.TextMeshProUGUI` | `Txt` |
| ✓ | `UnityEngine.UI.Image` | `Img` |
| ✓ | `UnityEngine.CanvasGroup` | `CG` |

### 第三步：配置互斥规则（可选）

菜单 **Window › 界面工具 › 互斥规则**

以矩阵形式配置：当节点同时含有多个组件时，高优先级组件存在则跳过低优先级组件的导出。

例如：勾选 `Button → Image`，表示节点上存在 `Button` 时，`Image` 默认不导出（在预览面板中默认不勾选，但仍可手动启用）。

### 第四步：标记 Hierarchy 节点

在节点名前加标记符（默认字符见下表），告知插件如何处理该节点及其子树：

| 标记 | 行为 |
|:---:|---|
| `@` | 导出此节点的组件为字段，不影响子节点 |
| `#` | 跳过此节点的组件，不影响子节点，子节点继续遍历 |
| `=` | 导出此节点，并让子树中无标记节点默认也导出 |
| `-` | 跳过此节点，并让子树中无标记节点默认也跳过 |
| 无标记 | 使用最近祖先 `=`/`-` 传播的默认值，若无则默认等同 `@` |

**示例 Hierarchy：**

```
ConfirmForm               ← 根节点（无标记，默认导出）
  =Panel                  ← 传播-导出：自身及子树默认导出
    #Background           ← 仅跳过自身，子节点继续继承 =Panel 的默认
    -TempBtnGroup         ← 传播-跳过：自身及子树默认跳过
      @BtnSure            ← 自身声明 @，免疫父节点的 - 传播，强制导出
      BtnCancel           ← 无标记，继承 -TempBtnGroup → 跳过
    TxtTitle              ← 无标记，继承 =Panel → 导出
    TxtContent            ← 无标记，继承 =Panel → 导出
```

> **节点自身声明的标记优先级最高**，不被任何祖先的 `=`/`-` 覆盖。

### 第五步：执行导出

在 **Project 窗口**中右键点击 UIForm 预制件 `.prefab`：

**Assets › Tools › 更新UI组件字段**

插件将：

1. 验证预制件路径和 UIForm 基类组件是否合法
2. 按标记规则遍历 Hierarchy，收集待导出组件
3. 打开预览面板，展示树形结构供用户二次调整
4. 用户确认后，生成/覆盖 `{ClassName}.Components.cs`，并在主脚本自动补全 `partial`
5. 触发 `AssetDatabase.Refresh()`，编译完成后自动将组件实例绑定到字段并保存预制件

## 生成代码示例

### 主脚本（首次导出后自动添加 `partial`）

```csharp
// ConfirmAForm.cs
namespace SYGameScript
{
    public partial class ConfirmAForm : UGuiForm<ConfirmAFormData>
    {
        // ... 业务逻辑
    }
}
```

### 自动生成的组件文件

```csharp
// ConfirmAForm.Components.cs  ← 自动生成，请勿手动修改
using TMPro;
using UnityEngine.UI;

namespace SYGameScript
{
    public partial class ConfirmAForm
    {
        // <auto-generated-components>
        public Button BtnSure;
        public Button BtnCancel;
        public TextMeshProUGUI TxtTitle;
        public TextMeshProUGUI TxtContent;
        // </auto-generated-components>
    }
}
```

> 再次执行"更新UI组件字段"时会**覆盖**整个 `<auto-generated-components>` 区域，业务逻辑写在主脚本中不受影响。

## 导出设置

菜单 **Window › 界面工具 › 导出设置**

可自定义 4 个标记符（每个必须是单个非空白 ASCII 字符，四者不能相同），点击 **重置为默认值** 恢复 `@ # = -`。

## 标记符说明（默认）

```
节点自身声明的标记  >  最近祖先 =/-  传播的默认值  >  全局默认 @

@ — 仅导出此节点的组件为字段
# — 仅跳过此节点的组件，子节点照常遍历
= — 导出此节点，同时让子树中无标记节点默认导出
- — 跳过此节点，同时让子树中无标记节点默认跳过

解析规则：取节点名第一个字符进行匹配，区分大小写。
未命中任何标记则视为无标记，使用继承默认值。
```

---

## 注意事项

- **标记符字符选择**：`@ # = -` 均不会被中文输入法全角替换，在中文 Windows 环境下安全使用
- **互斥规则不对称**：A→B 与 B→A 是独立配置，请在互斥面板中分别勾选
- **`Components.cs` 勿手动修改**：每次导出都会完整覆盖该文件；自定义逻辑请写在主脚本中
- **重试机制**：若编译分批完成导致绑定失败，插件最多自动重试 5 次 `delayCall`，仍失败时会在 Console 输出警告
